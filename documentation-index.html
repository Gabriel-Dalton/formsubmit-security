URL Filtering

Block specific web addresses from being submitted with your form to keep things secure.

Prevent certain URLs from being included in form submissions to ensure that only trusted links are accepted.

Implement URL filtering to block specific domains or URLs in your form submissions, ensuring only valid and approved URLs are processed.

Strategically manage URL submissions by whitelisting or blacklisting specific URLs, fortifying your form against unauthorized or malicious entries.

Adding the URL Ban List Field

To ban specific URLs from being included in form submissions, include the following HTML input element in your form:

<input type="hidden" name="_url_banlist" value="spammyurl.com, anotherbadurl.com, suspicioussite.org">
Customizing Your URL Ban List

You can customize the URL ban list to better fit your needs by adding URLs specific to the spam content you encounter. Here’s how you can update the URL ban list:

  1. Identify common spam URLs in the emails you receive.
  2. Add these URLs to the value attribute of the _url_banlist input element, separated by commas.
<input type="hidden" name="_url_banlist" value="spammyurl.com, anotherbadurl.com, newsuspicioussite.net">
Banning All URLs

To ban all URLs in form submissions, include the following HTML input element in your form:

<input type="hidden" name="_url_ban_all" value="true">

To allow only one specific URL, use the following HTML input element in your form:

<input type="hidden" name="_url_allow" value="https://allowedurl.com">

Content Security Policy (CSP)

CSP helps protect your website by controlling what resources (like scripts and styles) can be loaded, preventing harmful content from running on your site.

By implementing a Content Security Policy, you can specify which sources your website is allowed to load scripts, styles, and other content from, helping to prevent unauthorized access and attacks.

Content Security Policy is an essential security measure that restricts the sources from which your website can load resources, effectively blocking unauthorized scripts from executing and protecting against cross-site scripting attacks.

A well-crafted CSP is a critical component in your security architecture, ensuring that your web forms are not only functional but also resilient against a wide range of potential security threats, thus safeguarding your digital ecosystem.

Example Implementation:

To add a CSP to your form, include the following meta tag in the <head> section of your HTML:

<meta http-equiv="Content-Security-Policy" content="
    default-src 'self'; 
    script-src 'self'; 
    style-src 'self' https://fonts.googleapis.com; 
    font-src 'self' https://fonts.gstatic.com; 
    connect-src 'self';">
Benefits:
  • Enhanced Security: Prevents unauthorized scripts from executing on your form page, protecting against XSS attacks.
  • Controlled Resource Loading: Ensures that only resources from trusted sources are loaded, reducing the risk of data breaches.
  • Monitoring Violations: You can set up a reporting mechanism to monitor any CSP violations, allowing you to detect and respond to potential security issues in real-time.
Reporting Violations:

For advanced monitoring, you can add a report-uri directive to your CSP. This will send violation reports to a specified endpoint, helping you keep track of any attempts to bypass your security settings.

<meta http-equiv="Content-Security-Policy" content="
    default-src 'self'; 
    script-src 'self'; 
    style-src 'self' https://fonts.googleapis.com; 
    font-src 'self' https://fonts.gstatic.com; 
    connect-src 'self';
    report-uri https://yourdomain.com/csp-report-endpoint;">

This feature adds a layer of security to your FormSubmit forms, ensuring that your form remains secure from common web vulnerabilities while maintaining necessary functionality.
